A short reference for the indexing convention used by `ReelSetBuilder.initialFrame` and `ReelSet.setResult`. Read this once and keep it bookmarked when you start pre-filling buffer cells.

## Why buffers exist

Each reel keeps a few hidden cells **above** the visible window and a few **below** it. They scroll into view briefly during spins, hold the next symbol that will wrap onto the strip, and let win animations pan upward or downward without revealing empty strip. The count is set globally with `.bufferSymbols(n)` on the builder.

Most slots use `bufferSymbols(1)` or `bufferSymbols(2)`. Numbers higher than `2` are unusual and mostly useful for long deceleration arcs or cross-symbol presentations.

## Index-to-cell map

For a single column on a reel with `bufferAbove = 2`, `visibleRows = 3`, `bufferBelow = 1` (total strip length: 6 cells):

```
┌──────────────┬──────────────────┬───────────────────────────────────┐
│ frame[col][] │  Cell            │ Visible to player at rest?        │
├──────────────┼──────────────────┼───────────────────────────────────┤
│ [-2]         │  buffer-above 1  │ no — furthest above the mask      │
│ [-1]         │  buffer-above 0  │ no — closest above the mask       │
│  [0]         │  visible row 0   │ yes — top                         │
│  [1]         │  visible row 1   │ yes — middle                      │
│  [2]         │  visible row 2   │ yes — bottom                      │
│  [3]         │  buffer-below 0  │ no — closest below the mask       │
└──────────────┴──────────────────┴───────────────────────────────────┘
```

The buffer cells are hidden behind the viewport mask at rest. They become visible briefly during a spin when the strip scrolls them past.

## Form 1 — negative-index array (legacy, ergonomic one-liner)

```ts
const grid: string[][] = [
  ['A', 'B', 'C'],
  ['A', 'B', 'C'],
];
grid[0][-1] = 'COIN';     // closest above visible on reel 0
grid[0][-2] = 'WILD';     // furthest above on reel 0
grid[1][3]  = 'SCATTER';  // closest below visible on reel 1

reelSet.setResult(grid);
// or — same shape, build-time seed:
builder.initialFrame(grid);
```

This works because `arr[-1] = 'X'` creates a string property `"-1"` on the array object. The engine special-cases negative-index access in the frame-builder pipeline so the convention behaves exactly like the symmetric `arr[visibleRows]` form for buffer-below.

**Caveat:** standard cloning operators (`[...arr]`, `structuredClone`, `JSON.stringify`, `postMessage`) **drop** these properties because they iterate numeric indices only. The engine preserves them internally, but if your code passes the grid through a worker or serializer, use the explicit form below.

## Form 2 — explicit `ColumnTarget` (recommended for serialized code)

```ts
import type { ColumnTarget } from 'pixi-reels';

const grid: ColumnTarget[] = [
  {
    visible: ['A', 'B', 'C'],
    bufferAbove: ['COIN', 'WILD'], // [0] closest, [1] furthest
  },
  {
    visible: ['A', 'B', 'C'],
    bufferBelow: ['SCATTER'],      // [0] closest below
  },
];

reelSet.setResult(grid);
// or — same shape, build-time seed:
builder.initialFrame(grid);
```

Survives any clone, serializes cleanly to JSON, and TypeScript narrows on the field names rather than the negative-key access pattern. Both `setResult` and `initialFrame` accept either form interchangeably.

## Which slots are honoured where

| API | Visible (`[0..n-1]`) | Buffer-above (`[-k]` or `bufferAbove[]`) | Buffer-below (`[n..]` or `bufferBelow[]`) |
| --- | --- | --- | --- |
| `ReelSetBuilder.initialFrame` | yes | yes | yes |
| `ReelSet.setResult` (legacy) | yes | yes | yes |
| `ReelSet.setResult` (ColumnTarget[]) | yes | yes | yes |
| `Reel.placeSymbols` (low-level) | yes | yes | yes |

Random fill applies to any unset slot — buffer cells get a separate "buffer" pass from the symbol provider, so buffer-only symbol weights are configurable separately from visible-area draws.

## When to use which form

Reach for the **negative-index form** when:
- You are building the grid in the same process that calls `setResult`
- The grid is short-lived and won't pass through workers, postMessage, or `JSON.stringify`
- You want a one-liner: `grid[col][-1] = 'COIN'` is hard to beat for ergonomics

Reach for the **explicit `ColumnTarget` form** when:
- The grid crosses a worker boundary, a network call, or anything that serializes
- You want stronger TypeScript narrowing on which slots are set
- A reviewer would be more confused than helped by `arr[-1] = 'X'`
