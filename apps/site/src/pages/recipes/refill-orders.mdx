Each canvas below runs the same scripted spin: 6×4 cards land left-to-right with a pre-wired 3-cluster of `10` on row 2. The winners go through the same `destroyWinners` implode effect. Only the **refill drop-in order** varies between canvases — controlled by `setDropOrder()` on the reel set and `rowStagger` / `rowOrder` on the builder's `.tumble({ dropIn })` config.

Most commercial tumble slots use **simultaneous** for refills. The other three are useful for specific themes (puzzle, narrative, premium) — but all four read as "the new symbols arrived" without you writing motion code.

## Simultaneous — the canonical default

<RecipeRunner code={simultaneousCode} client:only="react" />

```ts
.tumble({
  dropIn: { rowStagger: 0, distance: 'perHole' },
})
// ...
reelSet.setDropOrder('all');  // 0 ms delay between reels
await reelSet.refill({ winners, grid: stage1 });
```

No in-reel row stagger (`rowStagger: 0`), no cross-reel delay (`setDropOrder('all')`). Every cell that needs to move starts moving at the same frame. Snappy, no pacing overhead — the player gets back to a "ready" grid as fast as the drop-in animation runs. This is what 80%+ of commercial cascade slots ship.

## Left-to-right wave

<RecipeRunner code={ltrCode} client:only="react" />

```ts
.tumble({
  dropIn: { rowStagger: 0, distance: 'perHole' },
})
// ...
reelSet.setDropOrder('ltr', 90);  // 90 ms between reels, left to right
await reelSet.refill({ winners, grid: stage1 });
```

Each reel's rows still arrive together, but reel 0 lands first, then reel 1, then reel 2... Reads as the grid "filling up" column-by-column. Visually distinctive but it adds noticeable time to every cascade — picture six cascades in a chain × 6 columns × 90 ms = nearly 3 extra seconds across the round.

Use when the cascade chain is rare (deep features, bonus rounds) or when the column-by-column reveal IS the slot's hook.

## Bottom-up row arrival

<RecipeRunner code={bottomUpCode} client:only="react" />

```ts
.tumble({
  dropIn: {
    rowStagger: 90,
    rowOrder: 'bottomToTop',
    distance: 'perHole',
  },
})
// ...
reelSet.setDropOrder('all');  // all reels at once
await reelSet.refill({ winners, grid: stage1 });
```

All reels drop simultaneously, but within each reel the bottom row lands first and the top row lands last. Reads as the board "stacking up from below". Pairs well with puzzle / match-3 / chess-board themes where the slot's metaphor is a physical board being built.

## Top-down row arrival

<RecipeRunner code={topDownCode} client:only="react" />

```ts
.tumble({
  dropIn: {
    rowStagger: 90,
    rowOrder: 'topToBottom',  // default — could be omitted
    distance: 'perHole',
  },
})
// ...
reelSet.setDropOrder('all');
await reelSet.refill({ winners, grid: stage1 });
```

The natural "rain from above" drop-in. Top row arrives first, bottom row last. With `distance: 'perHole'` the top-row new symbols actually fall the shortest distance — but the per-row stagger reverses the perceived order so the player reads "new symbols pour in from the top."

## Picking the refill order

| Refill | When it fits | Avoid when |
|---|---|---|
| **Simultaneous** | Default; you're not sure | — |
| **Left-to-right wave** | Slow cadence is fine; the wave IS the visual | Deep cascade chains (the per-spin overhead compounds) |
| **Bottom-up** | Puzzle / match-3 / chess-board themes | Naturalistic / gravity-themed slots |
| **Top-down** | Naturalistic, "rain from above" theme | When you want the lowest-effort refill (use Simultaneous instead) |

`setDropOrder` + `rowStagger` + `rowOrder` compose freely. A column-wave fall AND a per-row stagger AND a non-default rowOrder all coexist — the drop-in timeline lays them out additively.
