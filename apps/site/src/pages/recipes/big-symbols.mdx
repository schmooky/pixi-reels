A "big symbol" â€” a 2Ă—2 bonus frame, a 3Ă—3 giant â€” occupies multiple grid cells but represents a single icon. The engine paints the block from one anchor cell; you don't change the wire format.

## The whole mechanic

Register with size metadata:

```ts
.symbolData({
  bonus: { weight: 1, zIndex: 5, size: { w: 2, h: 2 } },
})
```

Server response stays `string[][]`. Place the symbol id at the **anchor cell** (top-left of the block) only. The engine fills the rest:

```ts
// 5Ă—4 grid, 2Ă—2 bonus at (col=2, row=1):
setResult([
  ['low1', 'low2', 'low1', 'low2'],
  ['low1', 'low2', 'low1', 'low2'],
  ['low1', 'bonus', 'X',    'low2'],   // 'X' is whatever; engine paints OCCUPIED
  ['low1', 'Y',     'Z',    'low2'],   // 'Y','Z' ignored â€” engine paints OCCUPIED
  ['low1', 'low2', 'low1', 'low2'],
]);
```

The 2Ă—2 anchor symbol is sized to `(2 Ă— cellW, 2 Ă— cellH)` and visually covers the whole block. `getVisibleSymbols()` and `getVisibleGrid()` resolve OCCUPIED cells to the anchor's id, so all four cells of a 2Ă—2 read as `'bonus'`.

## Anchor lookup

`reelSet.getSymbolFootprint(col, row)` returns the anchor + size for any cell â€” anchor or non-anchor:

```ts
const fp = reelSet.getSymbolFootprint(col, row);
// â†’ { anchor: { col: 2, row: 1 }, size: { w: 2, h: 2 } }
```

For the pixel rect covering the whole block, use `getBlockBounds`:

```ts
const rect = reelSet.getBlockBounds(col, row);
// â†’ { x, y, width: 2 * cellW, height: 2 * cellH }

gfx.rect(rect.x, rect.y, rect.width, rect.height)
   .stroke({ color: 0xff6b35, width: 4 });
reelSet.addChild(gfx);
```

For 1Ă—1 cells, `getBlockBounds` is equivalent to `getCellBounds`.

## Validation

The engine throws at `setResult()` if a block doesn't fit:
- "block exceeds reel height" â€” anchor's row + `size.h` > `visibleRows`.
- "block exceeds reel count" â€” anchor's col + `size.w` > `reelCount`.

Pinning a non-anchor cell of a big symbol throws â€” pin the anchor instead, which covers the whole block visually because the anchor's view spans the block.

## Mask strategy: use `SharedRectMaskStrategy` if reels have horizontal gaps

If `symbolGap.x > 0`, the default per-reel mask (one clip rect per reel) will **clip a cross-reel big symbol at every column gap** â€” visible as thin transparent vertical strips through the symbol. The fix is to switch to `SharedRectMaskStrategy`, which draws a single bounding rect:

```ts
import { SharedRectMaskStrategy } from 'pixi-reels';

builder
  .symbolGap(4, 4)                                  // horizontal gap > 0
  .maskStrategy(new SharedRectMaskStrategy())       // big symbol stays whole
  // ...
```

The recipe demo above does this. If your slot has zero horizontal gap (`symbolGap.x === 0`), you don't need to change anything â€” the per-reel rects are contiguous so big symbols render correctly.

## Constraints

- **Big symbols must have `weight: 0`.** Random fill can't place blocks in v1, so a non-zero weight would silently never get picked. The builder throws to surface the misunderstanding. Place big symbols via `setResult()` only.
- **Big symbols + MultiWays** is rejected at build. The 2Ă—2-on-a-2-row question is a game-design call.
- **Big symbols only at landing** â€” during the spin, every cell is 1Ă—1. The 2Ă—2 layout commits at stop.
- **No random-fill big symbols in v1** â€” only target frames (server-driven) place them. v2 may add a frame middleware that places big symbols during scroll.

## How OCCUPIED works internally

The cross-reel coordinator runs in `SpinController` ahead of per-reel `FrameBuilder`. It reads the result grid, finds big-symbol anchors, validates fit, and paints an internal `OCCUPIED_SENTINEL` across the rest of the block. Per-reel frame building leaves OCCUPIED cells alone (they're non-empty strings, so `RandomFillMiddleware` skips them). At land, an invisible `OccupiedStub` (an internal `ReelSymbol` with `view.alpha = 0`) sits at the OCCUPIED cell â€” the anchor's resized view covers it visually.

The sentinel never crosses the public API. Both `Reel.getVisibleSymbols()` and `ReelSet.getVisibleGrid()` resolve OCCUPIED â€” same-reel and cross-reel â€” to the anchor's id. Iterating per reel returns the same data as the grid surface.
