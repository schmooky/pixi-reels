A cascade/tumble slot on the engine. `.cascade()` replaces the strip-spin mechanic with a drop-in: symbols fall from above the viewport when the result arrives, with configurable per-row stagger and easing. The cascade loop itself (cluster detection, survivor shift, refill) stays in your code.

```ts
import { ReelSetBuilder, SpriteSymbol, DropRecipes } from 'pixi-reels';

const reelSet = new ReelSetBuilder()
  .reels(6)
  .visibleSymbols(5)
  .symbolSize(110, 110)
  .symbols((r) => {
    for (const id of SYMBOLS) r.register(id, SpriteSymbol, { textures });
  })
  .cascade(DropRecipes.stiffDrop)     // row-by-row drop, hard stop (no bounce)
  .ticker(app.ticker)
  .build();

app.stage.addChild(reelSet);

let multiplier = 1;
document.getElementById('spin')!.addEventListener('click', async () => {
  multiplier = 1;

  // Initial drop: columns land left-to-right
  const p = reelSet.spin();
  reelSet.setDropOrder('ltr');
  reelSet.setResult(await server.spin());
  let grid = (await p).symbols;

  while (true) {
    const wins = detectClusterWins(grid);     // your function
    if (wins.length === 0) break;

    await reelSet.spotlight.cycle(
      wins.map((w) => ({ positions: w.positions })),
      { displayDuration: 700 },
    );

    multiplier++;
    showMultiplier(multiplier);

    // Cascade refill: all columns drop simultaneously
    const next = tumble(grid, wins);          // remove + fall + refill
    const promise = reelSet.spin();
    reelSet.setDropOrder('all');
    reelSet.setResult(next);
    grid = (await promise).symbols;
  }
});
```

## Drop order controls the reveal feel

`setDropOrder` is a per-spin call — mix orders freely:

```ts
reelSet.setDropOrder('ltr');   // left → right (initial spin reveal)
reelSet.setDropOrder('rtl');   // right → left
reelSet.setDropOrder('all');   // all columns simultaneously (cascade refills)
reelSet.setDropOrder([0, 0, 150, 150, 300, 300]); // custom ms per reel
```

## Drop recipes

`DropRecipes` ships three presets you can pass directly to `.cascade()`:

| Preset | rowDelay | dropDuration | fallDuration | easing |
|---|---|---|---|---|
| `cascadeDrop` | 80 ms | 600 ms | 300 ms | `bounce.out` |
| `stiffDrop` | 60 ms | 450 ms | 280 ms | `power3.out` |
| `simultaneousDrop` | 0 ms | 500 ms | 300 ms | `bounce.out` |
| `slamDrop` | 0 ms | 0 ms | 0 ms | — |

## The cascade loop is your code

pixi-reels deliberately doesn't own the tumble rules — every game has quirks (cluster size, adjacency, minimum matches, sticky survivors). What you get from the library:

- **`.cascade()`** — drop-in mechanic, no strip spinning
- **`setDropOrder()`** — per-spin column reveal order
- **`setResult()`** on every tumble so each stage's grid is deterministic
- **`spotlight.cycle`** for the between-tumble win reveal

Your responsibilities:

- `detectClusterWins(grid)` — cluster/adjacency rules
- `tumble(grid, wins)` — compute the next-stage grid (surviving symbols + fresh fill)
- `showMultiplier(n)` — the UI meter
