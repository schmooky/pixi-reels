## The minimum code

```ts
import { WinPresenter, GraphicsLineRenderer } from 'pixi-reels';

const presenter = new WinPresenter(reelSet, {
  lineRenderer: new GraphicsLineRenderer(),
});

reelSet.events.on('spin:complete', async () => {
  const paylines = await server.wins(result);  // your server / eval
  await presenter.show(paylines);
});

// Cancel in-flight win presentation when a new spin starts.
reelSet.events.on('spin:start', () => presenter.abort());
```

## Payline shape

```ts
interface Payline {
  lineId: number;                 // stable id for per-line styling
  line: (number | null)[];        // row per reel, null to skip
  value: number;                  // amount (used for default sort)
  kind?: string;                  // optional tag routed to events
}
```

Per-reel `null` entries let you represent ways-to-win, cluster hits, or
partial lines without inventing a separate shape.

## What the default does

`WinPresenter` with its defaults:

- sorts paylines by `value` descending,
- fades non-winning symbols to `alpha: 0.35`,
- calls each winner's `playWin()` in parallel,
- renders a polyline through the cell centres (when a renderer is passed),
- cycles once with a 400 ms gap, fires `win:*` events throughout,
- restores symbol alpha on end.

## When to reach for a custom renderer

`GraphicsLineRenderer` is a prototype-grade polyline. For a premium look
(Spine line rig, dashed glow, per-line sprite sheet) implement the
[`LineRenderer`](/docs/) interface yourself and pass it as `lineRenderer`.

## UX guardrails

- Abort on `spin:start` — the snippet above.
- Disable the spin button between `win:start` and `win:end` if your UI can't slam-spin during celebrations.
- `cycles: -1` makes presentation loop until aborted — good for autoplay off-screens.
