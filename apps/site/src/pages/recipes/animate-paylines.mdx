## The minimum code

```ts
import { WinPresenter, type Win } from 'pixi-reels';

const presenter = new WinPresenter(reelSet, {
  stagger: 70,          // ms between cells — left-to-right sweep
  cycleGap: 350,        // ms between wins
});

reelSet.events.on('spin:complete', async () => {
  const wins: Win[] = await server.wins(result);   // your shape
  await presenter.show(wins);
});
reelSet.events.on('spin:start', () => presenter.abort());
```

## The whole API

```ts
interface Win {
  cells: SymbolPosition[];  // order matters when stagger > 0
  value?: number;           // used for default sort
  kind?: string;            // optional tag
  id?: number;              // optional stable id
}
```

One shape covers both classic paylines ("row 1 across all 5 reels") and
cluster pops ("these 7 cells vanished"). The presenter never distinguishes
them; it just animates the cells you hand it.

## Two knobs

| Option | Effect |
|---|---|
| `stagger: 0` | All cells in a win pulse together (a flash) |
| `stagger: 60+` | Cells animate one after another (a sweep). Pass cells in the order you want the sweep to run |
| `cycleGap: N` | ms between successive wins |
| `cycles: -1` | Loop forever until `abort()` |

## Paylines, outlines, numbers — draw them yourself

`WinPresenter` does not draw anything. To render a line over each payline,
subscribe to `win:group` and draw from `reelSet.getCellBounds`. See
[paylines-events-only](/recipes/paylines-events-only/) for a worked example.

## UX guardrails

- `abort()` on `spin:start`.
- `dimLosers: false` if your game already dims via an overlay.
- `stagger` tests well at 60–90 ms for a 5-reel sweep; shorter feels rushed, longer drags.
