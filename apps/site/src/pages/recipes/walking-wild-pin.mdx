Walking wild — the canonical "wild that moves" mechanic. Vikings Go Berzerk, Neon Staxx, Lil' Devil. Engine-native movement via `reelSet.movePin()` — no ghost sprites, no stage-level overlay.

Compare with the [original walking-wild recipe](/recipes/walking-wild/) which uses `new PIXI.Sprite(wildTex)` parented to the stage and tweens it above the reels. This version reparents a pooled `ReelSymbol` to the viewport's `unmaskedContainer` for the flight, then releases it back to the pool on landing.

## The whole mechanic

```ts
async function walkPinsLeft() {
  for (const pin of [...reelSet.pins.values()]) {
    if (pin.col <= 0) {
      reelSet.unpin(pin.col, pin.row);
      continue;
    }
    await reelSet.movePin(
      { col: pin.col, row: pin.row },
      { col: pin.col - 1, row: pin.row },
      { duration: 350 },
    );
  }
}

reelSet.events.on('spin:allLanded', ({ symbols }) => {
  // Capture new wilds that landed this spin
  for (let c = 0; c < symbols.length; c++) {
    for (let r = 0; r < symbols[c].length; r++) {
      if (symbols[c][r] === 'wild' && !reelSet.getPin(c, r)) {
        reelSet.pin(c, r, 'wild', { turns: 'permanent' });
      }
    }
  }
});
```

Call `walkPinsLeft()` between spins and you have a walking wild.

## What movePin does

1. **Atomic state update** — the pin map flips: `(from) → (to)` *before* the tween starts. Any subsequent spin sees the pin at its new home.
2. **Backfill the vacated cell** — the cell the pin left behind is filled with a random symbol from the frame builder's provider (or your `backfill` override).
3. **Reparent and tween** — a pooled `ReelSymbol` is acquired from the factory, parented to `viewport.unmaskedContainer` (above the mask so it can cross columns), positioned at the from cell, and tweened to the to cell via GSAP.
4. **Apply at destination** — when the tween completes, the pin's `symbolId` is placed at the destination cell via `placeSymbols`.
5. **Release the flight symbol** — back to the pool.
6. **Fire `pin:moved`** — with the new pin and the old coordinates.

## Options

```ts
reelSet.movePin(from, to, {
  duration: 350,              // ms, default 400
  easing: 'power2.inOut',     // any GSAP ease
  backfill: 'symbol-id',      // vacated cell filler; default = random
});
```

## Constraints

- Must be called at rest — throws if `isSpinning === true`. Movement happens *between* spins.
- Destination must be in-grid and unoccupied by another pin.
- `from === to` is a no-op that still fires `pin:moved` (so callers can treat the self-case uniformly).
