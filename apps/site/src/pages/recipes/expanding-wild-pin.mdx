Classic expanding wild — wild on land, whole column covered. Neon Staxx (NetEnt) and Wild Toro (ELK Studios) use this.

## The whole mechanic

```ts
reelSet.events.on('spin:allLanded', ({ symbols }) => {
  for (let c = 0; c < symbols.length; c++) {
    if (!symbols[c].includes('wild')) continue;
    for (let r = 0; r < symbols[c].length; r++) {
      if (symbols[c][r] !== 'wild') {
        reelSet.pin(c, r, 'wild', { turns: 'eval' });
      }
    }
  }
});
```

## Why `turns: 'eval'`

`'eval'` pins exist for exactly one spin's evaluation phase. They:

- Apply to the display immediately (the cell shows a wild)
- Survive through your game layer's win calculation and any animations
- Clear automatically when `spin:start` fires for the next spin

No cleanup code. The next spin is a clean slate.

## Not to be confused with book-style expanding

In a [book-style feature](/recipes/book-expanding-pin/), one symbol *class* is chosen at feature entry and expands whenever it appears. Expanding wild is per-cell: the wild at `(3,1)` expands its own column.
