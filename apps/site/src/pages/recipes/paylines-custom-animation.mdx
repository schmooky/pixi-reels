## The minimum code

```ts
const presenter = new WinPresenter(reelSet, {
  lineRenderer: new GraphicsLineRenderer(),
  symbolAnim: async (symbol, cell, payline) => {
    // Bigger bounce on the premium line, regular bounce elsewhere.
    const peak = payline.lineId === 0 ? 1.35 : 1.18;
    await bouncePulse(symbol.view, peak, 220);
  },
});
```

## Three modes in one field

`symbolAnim` accepts three shapes — pick whichever matches the asset:

| Value | Behaviour |
|---|---|
| `'win'` (default) | calls `symbol.playWin()` — your `ReelSymbol` subclass's hook |
| any other string | calls `symbol.playAnimation(name)` if present (e.g. `SpineSymbol`), else falls back to `playWin()` |
| `(symbol, cell, payline) => Promise<void>` | fully custom — you decide what to run |

## When to use this

- Sprite-based games that don't have spine animations — a GSAP scale/tint is the whole "win" gesture.
- Routing different paylines through different animations (scatter vs. line vs. way).
- Pairing a symbol animation with a sound cue keyed off `payline.value`.

The presenter still handles dim, line drawing, events, and cancellation — you only override the per-cell piece.
