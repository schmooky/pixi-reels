A common request: "the first round of the play should feel like a normal slot, but if there's a hit, the winners should pop and new symbols should drop in from above — like a tumble." This is exactly what the demo above does:

1. **Round 1** — a regular strip-spin lands a grid containing a winning cluster of three `10`s on the middle row.
2. **First cascade pop** — those three cells vanish, the symbol above each falls down to fill the gap, a brand-new symbol drops in at the top. The new top-row fill happens to create a second cluster (three `J`s on the bottom row).
3. **Second cascade pop** — those three `J`s vanish, survivors fall, new symbols drop in. No more wins, the play ends.

The reels never re-spin between cascades. The landed grid stays landed; only the cells involved in each pop move.

## How it works

The cascade animation is a separate concern from the spin. The library ships a tumble helper at `examples/shared/cascadeLoop.ts` that operates on the already-landed reelSet:

- It expects an array of "stages" — full grids representing the state after each cascade pop.
- It expects a `winners(prev, next, stageIndex)` callback that names the cells about to vanish.
- For each stage transition it runs your vanish animation (default: fade out), then slides the survivor symbols down by exactly the number of cleared slots beneath them, then drops new symbols in from above (staggered, gravity-style).

```ts
import { runCascade } from '../../shared/cascadeLoop';

await reelSet.spin();              // round 1, classic strip-spin
reelSet.setResult(stage0);
await /* wait for the spin to land */;

await runCascade(reelSet, [stage0, stage1, stage2], {
  winners: (_prev, _next, stageIdx) => {
    if (stageIdx === 0) return [{reel:1,row:1},{reel:2,row:1},{reel:3,row:1}];
    return [{reel:0,row:2},{reel:1,row:2},{reel:2,row:2}];
  },
  vanishDuration: 320,
  dropDuration: 440,
  pauseBetween: 160,
});
```

`runCascade` does NOT call `reelSet.spin()`. It works directly on the symbols already on screen — that's why the reels appear to stay still while the symbols rearrange themselves.

## When to use this vs `spin({ mode: 'cascade' })`

The library has two distinct cascade tools and they solve different problems:

| | `runCascade(reelSet, stages, ...)` | `reelSet.spin({ mode: 'cascade' })` |
|---|---|---|
| What runs | The visual tumble: pop, fall, drop-in | A full new spin using cascade phases (`dropStart` / `dropStop`) |
| Reels move? | No — landed reels stay landed; only winning + falling cells move | Yes — the whole frame drops in from above |
| Use case | Post-spin cascade animation on a single landed result (this recipe) | Discrete cascade respins between rounds (e.g. free-spin features that explicitly cascade-respin) |
| Builder requirement | None | `.cascade(DropRecipes....)` must be on the builder |

This recipe uses `runCascade` because the user-visible mechanic is "win on the spin, watch the winners pop". For "pure cascade respin" gameplay where every round is a top-down drop, see [Cascade 6×5 tumble](/recipes/cascade-6x5/).

## Full source

The exact code running in the demo above:

```ts
// @ts-nocheck
// Injected: ReelSetBuilder, SpeedPresets, CardSymbol, CARD_DECK, WILD_CARD,
//           DropRecipes, PIXI, gsap, app, runCascade, pickWeighted

const IDS = ['7', '8', '9', '10', 'J', 'Q'];
const REELS = 5, ROWS = 3, SIZE = 80;

function randSymbolNotIn(exclude) {
  let s;
  do { s = IDS[Math.floor(Math.random() * IDS.length)]; }
  while (exclude.has(s));
  return s;
}

const reelSet = new ReelSetBuilder()
  .reels(REELS).visibleSymbols(ROWS).symbolSize(SIZE, SIZE).symbolGap(4, 4)
  .symbols((r) => {
    for (const sym of CARD_DECK) {
      if (IDS.includes(sym.id)) {
        r.register(sym.id, CardSymbol, {
          color: sym.color, label: sym.label, textColor: sym.textColor,
        });
      }
    }
  })
  .speed('normal', { ...SpeedPresets.NORMAL, stopDelay: 120 })
  .ticker(app.ticker)
  .build();

return {
  reelSet,
  onSpin: async () => {
    // Stage 0: strip-spin lands here. Force a 3-of-'10' on row 1.
    const TRIGGER1 = '10';
    const HIT1_ROW = 1;
    const HIT1_COLS = [1, 2, 3];
    const stage0 = Array.from({ length: REELS }, (_, c) =>
      Array.from({ length: ROWS }, (_, r) =>
        r === HIT1_ROW && HIT1_COLS.includes(c) ? TRIGGER1 : randSymbolNotIn(new Set([TRIGGER1])),
      ),
    );

    // Stage 1: '10' cluster vanishes, survivors fall, new symbols fill.
    // The new fill creates a second cluster of three Js on row 2.
    const TRIGGER2 = 'J';
    const HIT2_ROW = 2;
    const HIT2_COLS = [0, 1, 2];
    const stage1 = stage0.map((col, c) => {
      const next = [...col];
      if (HIT1_COLS.includes(c)) {
        next[HIT1_ROW] = next[HIT1_ROW - 1];
        next[HIT1_ROW - 1] = randSymbolNotIn(new Set([TRIGGER1, TRIGGER2]));
      }
      if (HIT2_COLS.includes(c)) next[HIT2_ROW] = TRIGGER2;
      return next;
    });

    // Stage 2: 'J' cluster vanishes, survivors fall, new symbols fill.
    const stage2 = stage1.map((col, c) => {
      if (!HIT2_COLS.includes(c)) return [...col];
      const next = [...col];
      next[HIT2_ROW] = next[HIT2_ROW - 1];
      next[HIT2_ROW - 1] = next[HIT2_ROW - 2];
      next[0] = randSymbolNotIn(new Set([TRIGGER1, TRIGGER2]));
      return next;
    });

    // Round 1: classic strip-spin to stage 0.
    const p = reelSet.spin();
    await new Promise((r) => setTimeout(r, 150));
    reelSet.setResult(stage0);
    await p;
    await new Promise((r) => setTimeout(r, 300));

    // Cascade chain: pop + fall + fill, twice.
    await runCascade(reelSet, [stage0, stage1, stage2], {
      winners: (_prev, _next, stageIndex) => {
        if (stageIndex === 0) return HIT1_COLS.map((c) => ({ reel: c, row: HIT1_ROW }));
        return HIT2_COLS.map((c) => ({ reel: c, row: HIT2_ROW }));
      },
      vanishDuration: 320,
      dropDuration: 440,
      pauseBetween: 160,
    });
  },
};
```
